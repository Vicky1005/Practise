Для того чтобы сделать изменения нужно понимать на каком уровне эти изменения находятся.
От этого зависят варианты которые гит может предложить.

Working Directory:
1. git checkout --file.txt
git checkout .  - зачекаутить все файлы от текущей директории и ниже
! эта команда не имеет обратного эффекта (не вернуть, все пропало) - возврат до того состояния, которые известно гиту
git clean -xdf - удаление нового файла


Staging are (Index) - то есть после команды add
git reset --file.txt - убрать файл из индекса

Local Branch (как отменить комит)
git reset HEAD^^ или (HEAD~2) на сколько комитов назад нужно вернуться
git commit --amend -m "message" дописать
 
	меняет индекс	меняет файлы
в рабочей директории	               нужно быть
                                        внимательным
reset --soft	 нет	   нет	           нет
reset [--mixed]	  да	   нет	           нет
reset --hard	  да	   да	           да



Remote Repository stage - уже запушили изменения
git revert <sha1>
у каждого комита в гите есть свой  sha1
