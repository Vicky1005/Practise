static это поле класса, может стоять перед полем класса.
Для всех объектов класса статическое поле является фактически общим и имеет общее значение.
Если в одном из объектов значение этого статичексого поля будет изменено, то всем объектам класса будет доступно это изменение.
Статическое поле храниться отдельно от объекта и может существовать и использоваться даже если не существует ни одного объекта класса.
Статическое поле может обладать юбым модификатором видимости, инкапсуляция на него не накладывается. Если приватное, то гетеры и сетеры как правило на него не пишутся, 
через ккой-либо метод. (гетеры и сетеры плохая практика).
ДОступ к статичексому полю - через имя класса, можно через объект, но это плохая практика. (это нестатичское использование статического поля)

student1=null;
student1.faculty="geo";
System.out.println(student1.faculty);
//geo

Ошибки не будет так как компилятор сам вместо student1 подставит имя класса Student. Но это нехороштий кусок кода.


Блок Static - выполнение блока кода, конфигурирование логера, создание пула соединений для БД, либо онициализация статических полей.
блок static будет выхван самым первым независимо где бы он стоял в классе.
public class Student {

static{

faculty = "MMF";    //инициализируем статичексое поле по умолчанию
}

static String faculty;

}
